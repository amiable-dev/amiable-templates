# Support

This document describes how to get help with amiable-templates.

## Getting Help

### Documentation

Start with the documentation:
- [Quick Start Guide](https://amiable-dev.github.io/amiable-templates/quickstart/)
- [Template Documentation](https://amiable-dev.github.io/amiable-templates/templates/)
- [ADRs](https://amiable-dev.github.io/amiable-templates/adrs/)

### GitHub Discussions

For questions, ideas, and community support:
- [Q&A](https://github.com/amiable-dev/amiable-templates/discussions/categories/q-a) - Ask questions
- [Ideas](https://github.com/amiable-dev/amiable-templates/discussions/categories/ideas) - Suggest new templates or features
- [Show & Tell](https://github.com/amiable-dev/amiable-templates/discussions/categories/show-and-tell) - Share your deployments

### Issues

For bugs and specific problems:
- [Bug Reports](https://github.com/amiable-dev/amiable-templates/issues/new?template=bug_report.md)
- [Feature Requests](https://github.com/amiable-dev/amiable-templates/issues/new?template=feature_request.md)

## Template-Specific Support

For issues with specific templates, please report to the source repository:

| Template | Support |
|----------|---------|
| LiteLLM + Langfuse | [litellm-langfuse-railway](https://github.com/amiable-dev/litellm-langfuse-railway/issues) |
| LLM Council | [llm-council](https://github.com/amiable-dev/llm-council/issues) |

## What We Can Help With

- Documentation site issues (broken links, display problems)
- Template aggregation problems
- Questions about using the documentation
- Suggestions for improvements

## What We Cannot Help With

- Runtime issues with deployed templates (contact the template repo)
- LiteLLM or Langfuse application bugs (contact upstream)
- Railway platform issues (contact [Railway support](https://railway.app/help))
- General programming questions (try Stack Overflow)

## Response Times

- **GitHub Discussions**: Best effort, typically within a few days
- **Bug Reports**: Triaged within 7 days
- **Security Issues**: See [SECURITY.md](SECURITY.md) for SLAs

## Contributing

The best way to ensure your issue gets addressed quickly is to contribute a fix yourself! See [CONTRIBUTING.md](CONTRIBUTING.md) for guidelines.
