# Governance

This document describes the governance model for the amiable-templates project.

## Roles

### Users

Anyone who uses the documentation site or deploys templates based on our guides.

**Responsibilities:**
- Follow the [Code of Conduct](CODE_OF_CONDUCT.md)
- Report bugs and issues
- Provide feedback

### Contributors

Anyone who contributes to the project through pull requests, issues, discussions, or other means.

**Responsibilities:**
- Follow the [Contributing Guidelines](CONTRIBUTING.md)
- Follow the [Code of Conduct](CODE_OF_CONDUCT.md)
- Respond to feedback on contributions

### Maintainers

Contributors with merge access to the repository.

**Responsibilities:**
- Review and merge pull requests
- Triage issues
- Maintain documentation quality
- Ensure CI/CD health
- Follow security best practices

**Current Maintainers:**
- @amiable-dev/maintainers

### Core Team

Organization-level decision makers for amiable-dev projects.

**Responsibilities:**
- Set project direction
- Make architectural decisions (via ADRs)
- Manage maintainer access
- Handle security incidents
- Represent the project externally

## Decision Making

### Day-to-Day Decisions

- Made by maintainers through PR reviews
- Consensus preferred, majority acceptable
- If blocked, escalate to Core Team

### Architectural Decisions

- Documented as Architecture Decision Records (ADRs)
- Located in `docs/adrs/`
- Reviewed by LLM Council for significant decisions
- Require Core Team approval

### ADR Process

1. **Draft**: Author creates ADR with status "Draft"
2. **Review**: Submitted for LLM Council review (optional but recommended)
3. **Proposed**: Updated with council feedback, status changed to "Proposed"
4. **Discussion**: Open for community feedback via PR
5. **Accepted/Rejected**: Core Team makes final decision

## Becoming a Maintainer

Contributors may be invited to become maintainers based on:

- Sustained, quality contributions over 3+ months
- Demonstrated understanding of project goals
- Positive community interactions
- Reliability in responding to reviews

To express interest, open a Discussion or contact a current maintainer.

## Code of Conduct Enforcement

See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for enforcement guidelines.

## Changes to Governance

Changes to this governance document require:
- An ADR documenting the change
- LLM Council review (High tier)
- Core Team approval
